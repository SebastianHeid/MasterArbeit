\chapter{Experiments}
\label{sec:Experiments}
This chapter investigates low-rank model compression leveraging truncated \gls{SVD} on Transformer blocks within the PixArt-$\Sigma$ and FLUX.1 {[dev]} architectures. PixArt-$\Sigma$ is employed as an experimental baseline to evaluate various design choices in the compression framework. Its relatively low parameter count enables rapid iteration and extensive ablation studies. Subsequently, the most effective strategies are applied to FLUX.1 {[dev]} and FLUX.1 {[schnell]} to assess their efficacy in high-parameter regimes and in step-distilled models. 

During the experiments, only the \gls{DiT} backbone was subject to compression. The \gls{VAE}s and the text-encoders were kept unmodified.
\section{PixArt-$\Sigma$ Compression}
\label{sec:PixArtExp}

In the following sections, individual components of the compression framework are evaluated on PixArt-$\Sigma$ to identify the optimal configuration. To isolate the impact of each component, a default experimental setup is defined (Sec. \ref{sec:DefaultSetup}). In the subsequent experiments, only deviations from this default setting are explicitly mentioned to avoid repetition.

 The initial experiments investigate how to identify the model components that contain the highest redundancy and are therefore most suitable for compression (Sec. \ref{sec:ExpBlockImportanceAnalysis} and \ref{sec:PixartBlockSelectionAlgorithms}). Next,  two different compression schedules (Sec. \ref{sec:OneShot_vs_Progrosseve_Pruning}), the loss function (Sec. \ref{sec:ExpKDLoss}), and  post-compression fine-tuning protocols (Sec. \ref{sec:fine-tuningProtocol}) are evaluated. Furthermore, compression strategies, namely block pruning versus \gls{SVD}-based block compression are investigated (Sec. \ref{sec:CompressionStrategyPixart}). The focus then shifts entirely to \gls{SVD}-based compression to systematically analyze the sensitivity of Transformer block's components to low-rank approximation (Sec. \ref{sec:MLPAttnDiffCompRatios}), alongside compression schedules (Sec. \ref{sec:CompressionSchedulingPixart}). Finally, the found insights are summarized (Sec. \ref{sec:PixArtBestModel}) and leveraged in the following Sec. \ref{sec:ExpFlux} to compress FLUX.1 {[dev]}.

\subsection{Experimental Default Setup}
\label{sec:DefaultSetup}
This section defines the default configuration settings that are utilized in all experiments unless stated otherwise. It is important to note that this baseline configuration serves as a consistent starting point for the experiments and does not necessarily represent the final, optimal settings, which are derived later in this work. The baseline configuration comprises the following elements:

\begin{itemize}
		\item \textbf{Block Selection Algorithm}: The greedy algorithm based on \gls{CMMD} and \gls{CLIP}-score was utilized to identify and select the Transformer blocks for compression. The metrics were computed over 100 images, generated using JoyCaption prompts from the LAION-5B dataset.
		
	\item \textbf{Compression Schedule}: The progressive compression strategy was applied. As soon as the \gls{CMMD} value exceeded 0.1 during the block selection algorithm, the model was fine-tuned to recover its generation capabilities before the next compression iteration began.
	
	\item \textbf{Learning Objective}: Knowledge distillation was utilized for fine-tuning the compressed models. Therefore, the learning objective was defined as the sum of the unnormalized feature and output distillation losses,  $\mathcal{L_\text{OutKD}} + \mathcal{L_\text{FeatKD}}$.
	
	\item \textbf{Compression Strategy}: Structural pruning, specifically the complete removal of entire Transformer blocks, was employed to reduce the model size.
	
		\item  \textbf{Fine-tuning Protocol}: The recovery of the pruned model followed a structured three-stage fine-tuning protocol. First, for local stabilization, optimization was restricted to the block $B_{l-1}$ immediately preceding the removed block $B_l$ for one epoch (38k steps) on the LAION dataset. In cases where several blocks were removed simultaneously, all corresponding preceding blocks were updated while the rest of the network was frozen. Second, the complete model was optimized for an additional epoch on the LAION dataset. Third, for domain-specific refinement, the model was fine-tuned for two epochs (12.5k steps) on the LAION-PixArt dataset.  
		
		
		
	\item \textbf{Training Datasets}: Training was performed sequentially starting on the LAION dataset followed by further training on the LAION-PixArt dataset. For each iteration, the models were trained for two epochs on each dataset.

	\item \textbf{Evaluation}: The pruned models were evaluated on the HPSv2 \cite{HPSv2} and GenEval \cite{ghosh2023geneval} benchmarks. The HPSv2 benchmark assesses overall image quality by predicting a human preference score, whereas the GenEval benchmark analyzes text-image alignment.
	\item \textbf{Fine-Tuning Hyperparameters}: All other fine-tuning hyperparameters are summarized in Tab. \ref{tab:hyperparameterTraining}.

\end{itemize}

To visually illustrate the results, the main text typically displays images from only one compression level across the compared methods. The corresponding images for the other compression levels are provided in the Appendix.
The implementation of all experiments builds upon the official Git repository for PixArt-$\Sigma$ \cite{pixartsigma_code}.

\subsection{Block Importance Analysis}
\label{sec:ExpBlockImportanceAnalysis}
For model compression, the selection of components to be compressed is essential to ensure that the resulting smaller model retains the highest possible image generation quality. Qualitatively, Fig. \ref{fig:PixArtBlockImportanceQualitatively} shows the effect of removing each Transformer block from PixArt-$\Sigma$ individually on its image generation capabilities without subsequent fine-tuning. It is clearly visible that pruning the initial and final Transformer blocks (blocks 1-4 and blocks 27-28) as well as block 10 exerts the most significant influence on overall image quality. This confirms the observation from previous work \cite{men2024shortgpt} that redundancies are primarily found in the middle layers of Transformer-based models. To identify which blocks are most suitable for compression, i.e. those that make minimal contribution to the overall image generation process, several different metrics can be employed. In this section, various approaches are investigated to determine the importance of the individual Transformer blocks. First, traditional magnitude-based pruning and representational dissimilarity methods are evaluated, specifically by computing the centered kernel alignment score (\gls{CKA}) between the input and output of each Transformer block. Moreover, these methods are verified against two standard image quality metrics: \gls{CLIP}-score and \gls{CMMD}. 

 In practice, several Transformer blocks are compressed simultaneously. Due to block interdependencies, two different methods for identifying the best grouping of blocks are investigated. First, a greedy algorithm is applied, iteratively searching for the best next block to compress based on the already reduced model from previous iterations. Second, the block selection process is formulated as a hyperparameter search leveraging the Optuna framework.

\subsubsection{Block Selection Criteria}
\label{sec:ExpPixArtBlockSelectionCriterias}
In the magnitude-based pruning strategy, the model components with the lowest averaged weight magnitude are considered less important for overall model performance. Interestingly, in PixArt-$\Sigma$, the weights of the initial layers, which have a substantial influence on the final image synthesis, exhibit the lowest magnitudes across the architecture (Fig. \ref{fig:PixArtBlockImportanceMetrics} (b)). This observation contradicts the assumption that low weight magnitudes equate to low impact. It seems that despite their small weight magnitude, the first layers act as important initial feature translators, having a massive influence on model performance. Furthermore, the weight magnitudes of the final layers are only marginally higher than those of the middle layers, while the middle layers' weight magnitudes exhibit a high degree of similarity. This makes a precise selection of blocks for removal based solely on this heuristic difficult. Consequently, using weight magnitude to identify redundant blocks within PixArt-$\Sigma$ is not a suitable criterion in this context.

The \gls{CKA} score is computed for every block and every sampling step across 100 example prompts taken from the LAION dataset. Subsequently, these scores are averaged across all samples and time steps, resulting in a mean similarity score $\mu_{CKA,i}$ for each individual PixArt-$\Sigma$ block $B_i$. Averaging over the time steps should provide an estimate of the general importance of the block. However, the \gls{CKA} score can vary drastically across different timesteps (Appx. \ref{appx:BlockImpAnaPix}). A low value indicates that the input and output of a specific block differ significantly, suggesting a higher importance of the block. 

Fig. \ref{fig:PixArtBlockImportanceMetrics} (a) shows the transformation intensity $1-\mu_{CKA,i}$ for every block, a higher value signifies a more substantial feature transformation. While blocks 17 and 22 stand out with their high values, the implied importance of these blocks cannot be confirmed by qualitative analysis (Fig. \ref{fig:PixArtBlockImportanceQualitatively}). Furthermore, blocks 1, 2, 4, 27, and 28 are not uniquely identified as critical blocks for the performance of the model, as the values for blocks 19, 20, and 21 are higher, which would mark them as more relevant. As this ranking also fails to align with the qualitative review of the blocks' influence on the final images, the use of representational similarity for block selection in PixArt-$\Sigma$ must also be viewed critically.

Due to the contradictory results of the magnitude-based criterion and \gls{CKA}-scores when compared with the qualitative observations, a third methodology was investigated, namely quantifying the importance of individual blocks based on standard performance metrics \cite{FastFlux}. Specifically, the \gls{CLIP}-score and \gls{CMMD} were computed on a small reference dataset consisting of 100 images from the LAION dataset. It should be noted that the standard protocols for computing statistically robust \gls{CMMD} and \gls{CLIP}-scores typically require larger datasets (e.g. upwards of 1,000). However, given the iterative nature of the greedy search (Sec. \ref{sec:GreedyBlockSelection}) and the high computational overhead associated with repeated evaluations, using such a large reference set was prohibitively expensive. Therefore, a sample size of 100 images served as a sufficient proxy to capture the relative importance ranking of the Transformer blocks, providing the necessary directional guidance for the compression process. The effectiveness of this approach is demonstrated on FLUX.1 {[dev]} in Sec. \ref{sec:FluxParameterRemovalDistribution}.  As illustrated in Fig. \ref{fig:PixArtBlockImportanceMetrics} (c) and (d), both metrics closely align with the qualitative evaluation, confirming that the initial four and the last two blocks exert the greatest impact on the final image quality. Consequently, these metrics were selected as the primary criteria for identifying redundant blocks in PixArt-$\Sigma$.
This ranking serves as the foundation for the subsequent, more sophisticated block selection algorithm.

Furthermore, applying truncated \gls{SVD} to block 28 also results in less severe image quality degradation than completely removing it. This demonstrates that these blocks are critical for overall performance but also contain a high degree of redundancy. \subsection{Block Selection Algorithms}
\label{sec:PixartBlockSelectionAlgorithms}
A further key challenge after establishing the compression criterion is identifying the optimal combinations of blocks to reduce simultaneously. When compressing only a single block, there are merely 28 possibilities for PixArt-$\Sigma$. However, if the goal is to modify half of the Transformer blocks to obtain a strongly reduced model the number of possible block combinations escalates to $\binom{28}{14}$. Therefore, it is computationally infeasible to evaluate the \gls{CLIP}-score and \gls{CMMD} for every possible combination.

\subsubsection{Greedy}
A natural approach is to employ a greedy search algorithm. In this method, the importance ranking for all blocks is initially computed, after which the least important block is identified and compressed. Next, the selection criteria, e.g. \gls{CLIP}-scores and \gls{CMMD}, are re-evaluated for the remaining blocks within the newly reduced architecture. By iteratively evaluating and compressing the least important block, the model architecture is progressively reduced while accounting for the interdependencies between blocks. 

\subsubsection{Optuna}
Another approach is to reformulate the block selection task as a hyperparameter optimization problem, using the Optuna framework. The number of blocks $k$, the specific blocks $\{B_i\}_{i=1,\dots,k}$ to compress and their compression ratios $\{ \pi_i \}_{i=1,\dots,k}$ are potential hyperparameters to optimize. Here, the focus was on identifying the blocks $\{B_i\}_{i=1,\dots,k}$ given a fixed $k$ and  predefined $\{ \pi_i \}_{i=1,\dots,k}$ to reduce the search space. As the objective function, only the \gls{CMMD} metric was computed on 100 reference images based on prompts from the LAION dataset. To further speed up the search process, the results of the greedy algorithm were provided as a prior to the Optuna optimization framework, ensuring that the first trials are not picked randomly, which would drastically increase the search time until a good setting is found. 

The Optuna hyperparameter search was tested for the progressive SVD-based compression strategy with fixed compression ratios $\pi_i = 0.6$ $ \forall i \in \{1, \dots, k\}$, where the number of compressed Transformer blocks was chosen for each iteration individually. The procedure to determine the least important Transformer blocks is as follows. First, the greedy algorithm was leveraged to establish a prior of potentially good candidates for compression. Afterwards, the Optuna hyperparameter framework was applied to refine the block selection. In total, 300 different configurations were evaluated by Optuna, where the decision criterion was solely the \gls{CMMD} score. Once the optimal block combination was identified, the target blocks were compressed, and the resulting model was fine-tuned. This procedure was executed repeated for four iterations.

 Fig. \ref{fig:AdvantageOfOPtunaPfine-tuning} shows the prior blocks from the greedy algorithm as well as the final block selection of the Optuna algorithm. On the right side, the corresponding \gls{CMMD} values are displayed for each configuration. Note that these represent the \gls{CMMD} values before fine-tuning in order to compare the prior with the final selection. The grey blocks represent the consensus where the Optuna algorithm did not change the blocks from the prior, whereas the yellow and green blocks indicate the blocks exclusively selected by the greedy and Optuna algorithms, respectively. For iterations one to three, one observes that the majority of blocks from the greedy prior were maintained by the Optuna algorithm, only in iteration four do two out of three blocks differ. The large overlap is a first indication that the greedy algorithm already performs well. However, because the greedy result was used as a prior, the Optuna framework search is focused around it, which might also explain the high similarity. Overall, one can conclude that the \gls{CMMD} values obtained by the Optuna algorithm are consistently smaller (better) than the ones obtained by the greedy algorithm. This indicates that at least the pre-fine-tuning performance is enhanced by the refinement step.

Above, it is observed that in each iteration, the Optuna framework was able to identify a block combination for compression which resulted in a smaller \gls{CMMD} value compared to the prior greedy baseline.

To finally evaluate both the greedy and the Optuna approaches, their post-fine-tuning performances were compared. In this step, PixArt-$\Sigma$ was compressed twice, once based on the greedy and once based on the Optuna algorithm, each time targeting the same number of parameters (i.e. the same compression ratios $\pi_i = 0.6$ and the same number of blocks in each iteration). Fig. \ref{fig:AdvantageOfOPtunaPostTraining} shows the corresponding distribution of blocks compressed in each iteration. Please note that in this context "greedy" specifies the compression scheme where the greedy algorithm is strictly utilized as the sole block selection algorithm, in contrast to the previous section, where "greedy" served merely as the prior for the Optuna algorithm.

  An analysis of the block distribution reveals that the early layers are heavily compressed in both schemes, indicating a consensus on the high redundancy of these initial blocks. In the later layers, no clear structure is visible, and the selections diverge. This lack of a consistent pattern suggests that the deeper blocks are largely interchangeable regarding their impact on the model's capacity. 
  
 The evaluation of the fine-tuned compressed models (Fig. \ref{fig:BenchmarksOptuna}) does not provide clear evidence that the Optuna block selection algorithm leads to better-performing models post-fine-tuning. Specifically, for the smallest model, the HPSv2 score is even better when using the greedy algorithm, and a similar trend is observed for the second and third iterations regarding the GenEval score. This demonstrates that the fine-tuning process acts as an equalizer, effectively eliminating the small initial advantage of the Optuna algorithm. For this reason, the greedy algorithm is leveraged in the subsequent experiments as the additional computing time needed for Optuna is not justified by the results.

\subsection{One-Shot vs. Progressive Model Compression}
\label{sec:OneShot_vs_Progrosseve_Pruning}

One-shot model compression reduces the model to its target size in a single step, while progressive compression employs an iterative scheme of incrementally reducing parameters, followed by a recovery fine-tuning phase. Both frameworks were tested using otherwise the default settings for all other design choices (Sec. \ref{sec:DefaultSetup}). The number of blocks removed per model and the corresponding number of fine-tuning steps are summarized in Tab. \ref{tab:SettingOneShotvsProgressive}. 

%The exact blocks which are removed for each model can be found in the appendix \ref{sec:OneShotvsProgressiveBlockDistribution} (Fig. \ref{fig:OneShotvsProgressiveBlockDistribution}). 




Fig. \ref{fig:OneShotvsProgressive} displays the performance of one-shot versus progressive pruning on the HPSv2 and GenEval benchmarks across various compression ratios. To ensure a fair comparison, the models for both methods are fine-tuned for the same number of steps and on exactly the same data. Both benchmarks demonstrate that one-shot pruning is a viable strategy in the low-compression regime (10\% to 20\% parameter reduction). However, as the compression ratio increases, the progressive approach becomes advantageous because the incremental removal of parameters allows the model to recover its generative capabilities more effectively compared to the abrupt, large-scale parameter reduction in a single step. Fig. \ref{fig:Example_OneShot_vs_Progressive} presents visual evidence that the model progressively pruned to 50\%  of its original size generates images of higher quality than those obtained using the one-shot approach. Nevertheless, both approaches exhibit noticeable visual degradation at a compression ratio of 50\%. For visual results at other compression ratios, please refer to Appendix \ref{sec:OneShotVsProgressivePrunAppendix}.

\subsection{Knowledge Distillation Loss}
\label{sec:ExpKDLoss}
This section investigates the effectiveness of the individual components comprising the knowledge distillation loss (Eq. \ref{eq:KnowledgeDistillationLoss}) and their combinations. In total, five different loss functions were tested: 

 \begin{itemize}
 	\item \textbf{Output-Only Distillation} ($\bm{\mathcal{L_\textbf{OutKD}}}$): \\ The \gls{MSE} between the final output of the student and the teacher serves as the sole learning signal.
 	\item \textbf{Standard Distillation} ($\bm{\mathcal{L_\textbf{OutKD}}} + \bm{\mathcal{L_\textbf{FeatKD}}}$): \\ The unweighted sum of the final output loss and the  intermediate feature loss is leveraged as the learning objective.
 	\item   \textbf{Hybrid Distillation} ($\bm{\mathcal{L_\textbf{OutKD}}} + \bm{\mathcal{L_\textbf{FeatKD}}} + \bm{\mathcal{L_\textbf{Task}}}$): \\ The original diffusion loss is added to the standard distillation loss. 
 	\item \textbf{Normalized Distillation} ($\bm{\mathcal{L_\textbf{OutKD,norm}}} + \bm{\mathcal{L_\textbf{FeatKD,norm}}}$): \\ The final output and the intermediate feature loss are normalized (Sec. \ref{sec:KDLNormalization}) before being summed together to ensure that individual layers do not dominate the total loss. 
 	\item  \textbf{Normalized Hybrid Distillation} ($\bm{\mathcal{L_\textbf{OutKD,norm}}} + \bm{\mathcal{L_\textbf{FeatKD,norm}}}+ \bm{\mathcal{L_\textbf{Task}}}$): \\ This configuration extends the normalized distillation objective by incorporating the original diffusion task loss.
 \end{itemize}
 
 
 PixArt-$\Sigma$ was structurally pruned to varying sizes between 90\% and 50\% of the original model size utilizing the described loss configurations. Otherwise, the setup followed the default settings and the fine-tuning steps as described in Tab. \ref{tab:SettingOneShotvsProgressive}.  
 
 Fig. \ref{fig:LossInvestigation} presents the performance of the pruned models obtained by the different loss configurations. According to the HPSv2 benchmark, which measures overall image quality, the normalization of the intermediate features and the final output loss proves advantageous (blue and red lines in Fig. \ref{fig:LossInvestigation}). Both loss configurations that utilized normalization achieve a consistently higher HPSv2 score compared to their unnormalized counterparts. The normalization ensures that the contribution of each intermediate feature to the overall loss is balanced. This prevents the optimization process from prioritizing the later layers, where the errors are numerically larger, to the detriment of the earlier layers. The numerically larger discrepancies between the teacher and the student stem from the larger feature magnitudes in deeper layers, as displayed in Fig. \ref{fig:MagnitudeIntermediateFeatures}, which presents the \gls{RMS} of the intermediate features after every Transformer block in PixArt-$\Sigma$ averaged over 100 prompts. The increase in feature magnitude results from the signal accumulation inherent to the PixArt-$\Sigma$ architecture, which follows the residual formulation $\mathbf{x}_{i+1} = \mathbf{x}_i + B_i(\mathbf{x}_i)$, where $B_i$ represents the $i^\text{th}$ Transformer block. 
 
Of the two loss configurations that use normalization, the normalized hybrid distillation loss yields the better performance. This suggests that incorporating the original diffusion loss alongside the teacher's learning signal is advantageous for the fine-tuning process of the pruned models. The original loss might become critical when the teacher is uncertain itself about a prediction, as it anchors the student to the ground truth. 
 The benefit of the original diffusion loss is also confirmed by the unnormalized settings, where the hybrid distillation consistently achieves a higher HPSv2 score compared to the standard distillation approach.
 
   Interestingly, the output-only approach outperforms both standard and hybrid distillation settings for lightly pruned models. This suggests that the unbalanced features prevent effective learning. However, for strongly pruned models, the output-only loss configuration leads to an abrupt drop in quality. This shows that the intermediate feature loss, despite being unbalanced, serves as a critical stabilizer for the fine-tuning process in the high-compression regime.
  
  The GenEval benchmark largely corroborates the observations from the HPSv2 benchmark. The normalized hybrid distillation approach performs best overall, and adding the original diffusion loss consistently boosts performance. Notably, the output-only configuration results are even more pronounced, performing comparably well or better than the normalized hybrid distillation approach in the low-compression regime (90\% to 70\% of original model size). This suggests that at low compression, removing the constraints of feature matching allows the student to optimize semantic alignment very effectively. However, performance drops severely for larger compression rates. 
  Furthermore, it is worth noting that for the high-compression regime (60\% or 50\% models), the unnormalized hybrid distillation approach slightly outperforms the normalized distillation approach. This indicates that in the high-compression regime, the original diffusion loss is critical for maintaining text-to-image alignment.  
  
  
  
  
  Fig. \ref{fig:QualitativeLossInvestigation} presents a visual comparison between the five different loss function approaches, where PixArt-$\Sigma$ is pruned to 60\% of its original model size. When comparing the images based on pruning with different loss functions, it becomes clear that the facial contours (columns 1-3, 5) become distorted, especially in the output-only and standard distillation setups. In comparison, the images based on the normalized distillation and normalized hybrid distillation loss show more detail. The latter is also the only method in which both the horse and the astronaut are correctly displayed in the fourth column. All other methods generate only the horse, sometimes accompanied by significant artifacts. This indicates that the severe pruning weakened the text-image alignment, and the alternative loss functions were insufficient to recover it. Overall, these visual examples confirm the values from the performance benchmarks. Further example images of different compression ratios can be found in the Appendix \ref{sec:KnowledgeDistillationAppendix}.

 \subsection{Fine-Tuning Protocol}
 \label{sec:fine-tuningProtocol}
 
 Following the pruning of blocks from the PixArt-$\Sigma$ architecture, the reduced model was fine-tuned to recover its image generation capabilities. In total, three different fine-tuning protocols were tested to identify the most effective one. In all protocols, the model was trained for 88,500 optimization steps per iteration of the progressive pruning schedule to enable a fair comparison.
 \begin{itemize}
 	\item \textbf{Three-Stage Fine-Tuning Protocol} \\
 	This is the default fine-tuning protocol described in Sec. \ref{sec:DefaultSetup}. It is repeated here for the reader's convenience.
 	\begin{enumerate}
 		\item Local Stabilization: The blocks immediately preceding the removed blocks were fine-tuned for 38,000 steps on the LAION dataset, while all other parts of the model were frozen.
 		\item Global Optimization: The complete model was optimized for one epoch (38,000 steps) on the LAION dataset.
 		\item Domain-Specific Refinement: The LAION-PixArt dataset was leveraged to fine-tune the pruned model for 12,500 steps on its teacher's domain.
 	\end{enumerate}
 	
 	\item \textbf{Two-Stage Fine-Tuning Protocol} \\
 	In the following protocol, instead of performing local fine-tuning, the entire model was directly fine-tuned for 2 epochs on the LAION dataset.
 	\begin{enumerate}
 		 	\item Global Optimization: The complete model was optimized for two epochs (76,000 steps) on the LAION dataset.
 		\item Domain-Specific Refinement: The LAION-PixArt dataset was leveraged to fine-tune the pruned model for 12,500 steps on its teacher's domain.
 	\end{enumerate}

 	\item \textbf{Two-Stage LAION only Fine-Tuning Protocol} \\
 	The absence of the domain-specific refinement was investigated in this protocol.
 	\begin{enumerate}
 		\item Local Stabilization: The blocks immediately preceding the removed blocks were fine-tuned for 38,000 steps on the LAION dataset while all other parts of the model were frozen.
 		\item Global Optimization: The complete model was optimized for 50,500 steps on the LAION dataset to ensure the same number of training steps as in the previous fine-tuning protocols.
 	\end{enumerate}
 \end{itemize}

  	
 The performance evaluation of models across a pruning range from 90\% to 50\% of their original size can be found in Fig. \ref{fig:fine-tuningProtocols}. For both the HPSv2 and GenEval benchmarks, fine-tuning solely on the LAION dataset (yellow line) leads to a sharp decline for strongly compressed models. This strongly indication that the superior image quality of the LAION-PixArt dataset compared to the standard LAION dataset is critical for recovering of the generation capabilities of a pruned Transformer-based diffusion model. Moreover, it is observed that for lightly compressed models (90\% to 80\%), the two-stage fine-tuning protocol (green line) performs better than the three-stage protocol (purple line). This indicates that in the low-compression regime, the model benefits from the extended global fine-tuning, and no local repair is needed. For moderately compressed models (70\% to 60\%), the three-stage fine-tuning is consistently better than or on par with the two-stage protocol. This performance gap suggests that the local stabilization step provides necessary structural regularization when the model capacity is reduced to a critical threshold. For severe compression (50\%), there are contradictory results. According to the HPSv2 benchmark, the three-stage approach is still marginally better than the two-stage approach. However, the GenEval benchmark suggests that the reverse is true. Qualitatively, Fig. \ref{fig:QualitativeFinetuningProtocol} showcases example images from models compressed to 60\% using the three-stage, two-stage, and two-stage LAION-only fine-tuning protocols. Interestingly, the images from the three-stage protocol exhibit slightly fewer artifacts, suggesting that local stabilization is advantageous. However, since the quantitative metrics do not clearly favor one approach, the two-stage protocol presents a highly competitive baseline.

 \subsection{Compression Strategy}
 \label{sec:CompressionStrategyPixart}
  	
 As an alternative to the pruning strategy of removing entire Transformer blocks from \gls{DiT}s, low-rank approximation via truncated \gls{SVD} can be utilized to reduce the size of projection matrices within Transformer blocks. In PixArt-$\Sigma$, \gls{SVD}-based compression was applied to the \gls{MLP}, self-attention, and cross-attention matrices (Fig. \ref{fig:PixArt_Sigma_Compresseed_Modules_Architecutre}). To evaluate the efficiency of this compression strategy against structural block pruning, a representative operating point was chosen. The rank for the weight matrices within the \gls{MLP}s was set to $r_\text{MLP} = 512$, corresponding to a compression of approximately 44\% of its parameters. The ranks for the self- and cross-attention matrices were fixed to $r_\text{self-attn} = r_\text{cross-attn} = 128$, representing a parameter reduction of approximately 78\%. This served as a starting point. Later, the choice of compressing attention matrices more aggressively than the \gls{MLP} matrices is further investigated (Sec. \ref{sec:MLPAttnDiffCompRatios}). 

 
 
  Tab. \ref{tab:SVDCompressionFirstExp} compares the configurations of both methods, specifically the number of blocks removed or compressed and the corresponding training duration for each model. Note that the number of training steps differed significantly, as the default criterion - a \gls{CMMD} value exceeding 0.1 - was employed to determine the compression rate per iteration, consistent with the default settings (Sec. \ref{sec:DefaultSetup}). The observation that \gls{SVD}-based compression allows for larger parameter reduction with a less severe impact on the \gls{CMMD} value was an initial indication that low-rank approximation is a more nuanced method than block pruning. 
  
  Fig. \ref{fig:SVD_vs_Removal} presents the performance of both strategies on the HPSv2 and GenEval benchmarks. For both benchmarks, the \gls{SVD}-based compression method outperforms the complete block removal method in the low- and high-compression regimes. This suggests that distributing the parameter reduction across multiple blocks is less detrimental to model performance compared to removing individual blocks completely.
  For a qualitative comparison of both methods, Fig. \ref{fig:Example_Images_SVD_vs_Removal} provides a set of example images for models compressed to 50\% of their original size, which illustrates the advantage of truncated \gls{SVD} as a compression technique. The resulting images of this approach contain more details, exhibit fewer artifacts, and more closely resemble the original image. Specifically, column four highlights that the fish is more accurately preserved when applying the \gls{SVD} compression technique. Moreover, column one shows that in high-compression regimes, complete block removal can lead to severe deviations from the original image. For further visual examples produced by models of different sizes, refer to Appendix \ref{appx:CompressionStrategy}.
   
  In conclusion, the qualitative results confirm the quantitative benchmark performance and reveal a clear advantage of \gls{SVD}-based compression over block pruning, indicating that most blocks have an independent functional task that is difficult for neighboring blocks to fully recover, but that there are redundancies within the blocks themselves which can be effectively reduced by low-rank approximation.

 \subsection{Sensitivity Analysis of MLP and Attention Layers under \gls{SVD} Compression}
 \label{sec:MLPAttnDiffCompRatios}
 In this section, the \gls{SVD}-based compression strategy was applied, but otherwise the default settings remained. The goal was to investigate how to best distribute the parameter reduction within each Transformer block. Therefore, individual compression ratios were applied to the \gls{MLP} and attention matrices, whereby both self- and cross-attention matrices were compressed by the same amount. In total, three different scenarios were explored:
 
 \begin{itemize}
 	\item \textbf{Equal Compression}: \\
 	Both \gls{MLP} and attention matrices were compressed by the same percentage, $c_\text{MLP} = c_\text{Attn} = 60\%$. 
 	\item \textbf{Attention-Dominant Compression}: \\
 	The attention matrices were more strongly compressed ($c_\text{Attn} = 80\%$) compared to the \gls{MLP} matrices ($c_\text{MLP} = 40\%$).
 	 	\item \textbf{\gls{MLP}-Dominant Compression}: \\
 	The \gls{MLP} matrices were more strongly compressed ($c_\text{MLP} = 80\%$) compared to the attention matrices ($c_\text{Attn} = 40\%$).
 \end{itemize}
 
 Note that the combined parameter count of self- and cross-attention is approximately equal to the parameter count of the \gls{MLP}. Therefore, all configurations discarded a similar number of total parameters from one Transformer block. 
 
The scores in Fig. \ref{fig:DistributionCompressionRatios} reveal two distinct trends. The HPSv2 score demonstrates that prioritizing the compression of the attention matrices consistently yields higher image quality than the method focused on compressing the \gls{MLP} matrices. The GenEval benchmark, however, shows that although the attention-dominant approach is advantageous in the low-compression regime, it declines sharply under heavier compression, eventually falling below the \gls{MLP}-dominant approach for models with approximately 400 million parameters. 

 One possible explanation for the divergence between the two benchmarks is the distribution of tasks among the different components of a Transformer block. A recent study \cite{dong2025attention} suggests that \gls{MLP}s are crucial for knowledge storage and the processing of visual features in \gls{DiT}s. Consequently, strong compression of these layers leads to a significant decrease in image quality (HPSv2: yellow line).
  Fig. \ref{fig:ImagesAttnVsMLP} visually supports the result of the HPSv2 benchmark. The images clearly demonstrate that the \gls{MLP}-dominant approach lacks detail, texture, and structural integrity in the high-compression regime. For instance, the \gls{MLP}-dominant approach hallucinates an unprompted headband in the samurai image (column 4), and the facial features become distorted. Moreover,  the burger (column 2) is hovering in the air instead of being placed on the table, and the humanoid (column 5) is severely deformed. In contrast, the models of the attention-dominant approach, despite being approximately of the same size, do not indicate such severe structural errors, although a general degeneration of image quality at higher compression rates is also visible. 
  
  
 On the other hand, the GenEval benchmark measures text-image alignment. Since the cross-attention mechanism is responsible for integrating the text prompts into the visual latents, its strong compression in many Transformer blocks severely compromises this linking. Although the degradation in image quality is less severe compared to strong \gls{MLP} compression, the model's text comprehension deteriorates disproportionately. This explains why the attention-dominant approach loses its advantage for heavy compression and is in some cases even worse than the \gls{MLP}-dominant approach. 
 
 The equal compression strategy serves as a baseline compromise between the \gls{MLP}- and attention-dominant approaches. It exhibits results similar to the attention-dominant approach in the high-compression regime, but is slightly worse in the low-compression regime.

 \subsection{Importance-Aware Rank-Adaptive Progressive Model Compression}
 \label{sec:CompressionSchedulingPixart}


 
In previous experiments, SVD-based compression was applied to each block at most once using the progressive compression strategy (Sec. \ref{sec:MethodProgressiveModelPruning}). However, restricting the reduction to untouched blocks may be suboptimal. Incrementally reducing the size of an already modified block could potentially preserve model performance better than compressing a new, dense block. For this reason, the importance-aware rank-adaptive progressive model compression schedule defined in Sec. \ref{sec:LinearProgressiveCompression} was investigated. Assuming that a fixed amount of parameter reduction should be achieved per iteration, concretely 10\%, a key design choice is how to distribute this reduction across the model. The parameter reduction can either be distributed across many blocks lightly, or concentrated heavily within a few blocks. To determine which strategy works best, three different configurations with varying numbers of compressed blocks per iteration were evaluated:

\begin{itemize}
	\item \textbf{SVD\_7}: Seven blocks were compressed per iteration.
	\item \textbf{SVD\_14}: Fourteen blocks were compressed per iteration.
	\item \textbf{SVD\_21}: Twenty-one blocks were compressed per iteration.
%	\item \textbf{SVD\_28}: All blocks are compressed per iteration.
\end{itemize}
Tab. \ref{tab:PixArtAppendixCompression_iterations} provides detailed information about the base compression ratio and the linear decay rate $x$ for all three configurations.

For comparison, the \textbf{SVD\_Standard} baseline, where truncated \gls{SVD} was applied at most once per block leveraging the standard progressive compression schedule, was utilized.

Fig. \ref{fig:CompressionSchedule} presents the performance of the evaluated compression schedules on the HPSv2 and GenEval benchmarks.
According to the HPSv2 score, the differences in the low-compression regime are negligible, but they become more pronounced at higher reduction rates. 
 A significant performance drop for the \textbf{SVD\_7} schedule at a 50\% model size is observed, while the performance of \textbf{SVD\_14} and \textbf{SVD\_21} remains comparable. Furthermore, the \textbf{SVD\_Standard} baseline performs only slightly worse than these two configurations. This suggests that in order to maintain high image quality, it is advantageous to distribute the discarded parameters across the model instead of concentrating them on a few blocks.
 
 The GenEval benchmark demonstrates that in the high-compression regime, the advantages of \textbf{SVD\_14} and \textbf{SVD\_21} are more pronounced compared to \textbf{SVD\_Standard} and \textbf{SVD\_7}.
Therefore, it also seems beneficial for text-to-image alignment to allocate the compression budget more uniformly across the architecture. This observation is also qualitatively supported by the example images in Fig. \ref{fig:PruningSchedule}. For further visual examples of different compression stages, refer to Appendix \ref{appx:ImpAwRanAdProgComp}.

 \subsection{Summary of Best Settings for PixArt-$\Sigma$ Compression}
 \label{sec:PixArtBestModel}
The results of all experiments establish the following optimal configuration for \gls{DiT} compression:
 
 \begin{itemize}
 	\item \textbf{Block Importance Analysis} \\
 	Sec. \ref{sec:ExpBlockImportanceAnalysis} demonstrates that using image quality metrics on a small reference dataset as the primary block selection criterion is highly effective. Particularly in the importance-aware rank-adaptive progressive model compression scheme, where the number of compressed parameters varies per block, the \gls{CMMD} can be weighted by the number of reduced parameters to ensure a fair comparison. 
 	
 	When selecting a subset of Transformer blocks for simultaneous compression, the greedy algorithm proved highly effective, such that more advanced methods like Optuna do not provide substantial benefits that would justify the additional compute (Sec. \ref{sec:PixartBlockSelectionAlgorithms}).
 	
 	\item \textbf{Compression Schedule} \\
 	Progressive model compression proves advantageous over one-shot model compression in the high-compression regime, where models are compressed to below 70\% of their original size (Sec. \ref{sec:OneShot_vs_Progrosseve_Pruning}).
 	
 	\item \textbf{Knowledge Distillation Loss} \\
 	The results indicate that the combination of the original diffusion loss with the normalized feature and output loss leads to the strongest performance in all compression regimes (Sec. \ref{sec:ExpKDLoss}). 
 	
 	\item \textbf{Fine-Tuning Protocol} \\
 	The findings suggest that high image quality is essential to recover the image generation capabilities of compressed models. Therefore, incorporating the LAION-PixArt dataset into the training procedure was critical, demonstrated by the higher HPSv2 and GenEval scores for the three- and two-stage fine-tuning protocols. However, the results regarding the superiority of the three-stage protocol over the two-stage protocol are inconclusive. Based on the benchmark performance, it is not clear whether local fine-tuning is beneficial for high-compression regimes (Sec. \ref{sec:fine-tuningProtocol}). Although the three-stage approach may be preferred due to the visually more compelling images, the two-stage protocol remains a highly competitive baseline.
 	
 	\item \textbf{Model Compression Strategy} \\
 	The experiments demonstrate that utilizing truncated \gls{SVD} compression rather than pruning entire Transformer blocks leads to superior performance, even with fewer training steps (Sec. \ref{sec:StructuralCompressionStrategy}).
 	
 	\item \textbf{Sensitivity Analysis of MLP and Attention Layers under \gls{SVD} Compression} \\
 	The sensitivity analysis for \gls{MLP} and attention layers reveals that strong compression of \gls{MLP}s leads to more severe deterioration than heavy compression of attention layers. However, compressing both \gls{MLP} and attention layers equally yields performance comparable to the strong attention compression setup, though it performs slightly worse in the low-compression regime (Sec. \ref{sec:MLPAttnDiffCompRatios}).
 	
 	\item \textbf{\gls{SVD} Compression Schedule} \\
 	The results from Sec. \ref{sec:CompressionSchedulingPixart} suggest that importance-aware rank-adaptive progressive model compression performs better than standard progressive compression. Furthermore, distributing the parameter reduction over several blocks (e.g. 14 or 21) is more favorable than distributing it over fewer blocks, which consequently need to be compressed more heavily. As neither approach (14 or 21 blocks) proved clearly superior, targeting 21 blocks was chosen as the optimal configuration. 
 	
 \end{itemize}
All models resulting from this core importance-aware rank-adaptive progressive \gls{SVD} compression approach, including the adapted versions for FLUX.1 {[dev]} in the upcoming section, will henceforth be referred to as \textit{SVDtrunc}.
In the following section, these best practices are applied to FLUX.1 {[dev]}. Note that due to the different architecture, and therefore potentially different behavior, the sensitivity analysis could not be directly transferred to FLUX.1 {[dev]}. Additionally, because a different dataset was chosen, the fine-tuning protocol has also been slightly adapted.
